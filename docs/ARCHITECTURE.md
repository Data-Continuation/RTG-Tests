# RTG Architecture

## Core Model

Relative Transition Geometry treats a transition as a governed event moving through layered constraints.

A transition may be:

```text
claimed
observed
classified
admissible
authorized
replayable
receipt-consistent
lineage-consistent
exportable
finalizable
```

These are not the same thing.

## Primary Surfaces

```text
source identity surface
transition identity class
observer identity class
detector compatibility
observer window
admissibility pressure
authority class
receipt chain
state lineage
confidence posture
risk posture
quarantine posture
export boundary
master-record export posture
```

## Transition Rule

```text
transition claim
≠
valid transition evidence
```

## Observer Rule

```text
transition exists
≠
transition is governably observable
```

Observation requires identity-compatible coupling between observer and transition class.

## Authority Rule

```text
visible + stable + confident
≠
authorized to emit
```

Authority is an independent boundary.

## Replay Rule

```text
A transition claim must be replayable or receipt-consistent before it can be treated as valid state evidence.
```

## Cross-Layer Rule

No single layer is sufficient.

RTG becomes meaningful when layers are tested together:

```text
authority + replay
observer + detector
uncertainty + anomaly
supersession + lineage
quarantine + confidence
emission + governance output
receipt + export
commit + finality
```
